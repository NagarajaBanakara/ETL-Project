This project demonstrates a complete ETL (Extract, Transform, Load) pipeline using Python, Flask, and MySQL.
Additionally, a web UI is provided to allow users to insert employee data into the staging table, from where it will be processed into the final target table using the ETL script.

Components included:

| Component      | Technology                             |
| -------------- | -------------------------------------- |
| UI Application | Flask (Python)                         |
| Database       | MySQL                                  |
| ETL Script     | Python                                 |
| Data Flow      | UI → Staging Table → ETL → Final Table |


2. Architecture Flow

   +------------+       +------------------+       +-------------------+
   | Flask UI   | --->  | Staging Table    | --->  | ETL Python Script |
   | (Employee Form) |   | employee_staging |       | processed_position_data |
   +------------+       +------------------+       +-------------------+

3. Database Schema
Table 1: employee_staging

CREATE TABLE employee_staging (
    id INT AUTO_INCREMENT PRIMARY KEY,
    emp_id VARCHAR(20),
    emp_name VARCHAR(50),
    emp_dept VARCHAR(50),
    emp_salary DECIMAL(10,2),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);


Table 2: processed_position_data

CREATE TABLE processed_position_data (
    id INT AUTO_INCREMENT PRIMARY KEY,
    emp_id VARCHAR(20),
    emp_name VARCHAR(50),
    emp_dept VARCHAR(50),
    emp_salary DECIMAL(10,2),
    processed_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

5. ETL Script Functionality
| Step      | Operation                                         |
| --------- | ------------------------------------------------- |
| Extract   | Reads data from employee_staging                  |
| Transform | Trim, clean, validate, convert salary             |
| Load      | Inserts cleaned data into processed_position_data |
| Audit     | Prints logs + row count                           |

Run ETL
etl_flask_app.py

6. Logging & Error Handling

Try-except blocks used

Ensures DB rollback on failure

End time

Number of rows inserted
